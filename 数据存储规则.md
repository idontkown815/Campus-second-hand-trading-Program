# 数据存储规则

## 1. 数据库存储

### 1.1 存储方式
- **数据库类型**：MySQL
- **数据库名称**：`campus_trade`
- **用户名**：`root`
- **密码**：`123456`
- **主机**：`localhost`
- **端口**：`3306`
- **说明**：使用MySQL 8.0+版本作为数据库存储方案

### 1.2 数据库结构
- **用户表**：存储用户信息，包括学号、姓名、密码等
- **商品表**：存储商品信息，包括标题、描述、价格、图片等
- **分类表**：存储商品分类信息
- **交易表**：存储交易记录
- **通讯表**：存储用户通讯信息

## 2. 图片存储

### 2.1 存储方式
- **存储路径**：`uploads/` 文件夹
- **存储类型**：本地文件系统存储
- **说明**：项目上传的所有图片都存储在 `uploads` 文件夹中

### 2.2 图片访问机制
- **访问URL**：`http://localhost:8000/uploads/{图片文件名}`
- **前端显示**：通过完整的URL路径访问和显示图片
- **上传处理**：使用Django的FileSystemStorage进行文件存储

### 2.3 图片上传流程
1. 用户在前端选择图片文件
2. 前端通过API将图片上传到后端
3. 后端将图片保存到 `uploads` 文件夹
4. 后端返回图片的访问URL
5. 前端使用返回的URL显示图片

## 3. 配置说明

### 3.1 媒体文件配置
```python
# settings.py
MEDIA_ROOT = BASE_DIR / 'uploads'
MEDIA_URL = 'uploads/'

# 确保上传目录存在
import os
if not os.path.exists(MEDIA_ROOT):
    os.makedirs(MEDIA_ROOT, exist_ok=True)
```

### 3.2 URL配置
```python
# urls.py
if settings.DEBUG:
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
```

### 3.3 图片上传API
- **接口**：`POST /api/v1/products/upload/`
- **权限**：需要登录
- **参数**：`file` (图片文件)
- **返回**：图片的访问URL

## 4. 注意事项

1. **安全性**：上传的图片需要进行类型验证，防止恶意文件上传
2. **性能**：大图片需要进行压缩处理，提高加载速度
3. **备份**：定期备份数据库和图片文件
4. **清理**：定期清理过期的图片文件，释放存储空间

## 5. 未来扩展

- **云存储**：考虑使用阿里云OSS等云存储服务，提高可靠性和访问速度
- **CDN**：使用CDN加速图片访问
- **数据库优化**：根据业务需求，可能需要迁移到MySQL或PostgreSQL等关系型数据库
